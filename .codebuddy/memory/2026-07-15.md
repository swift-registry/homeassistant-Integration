# 2026-07-15

## homeassistant-Integration (扬州电费预估 HACS 集成)

- 仓库根因：HACS 只能添加 **github.com** 仓库；原仓库在自建 Git 服务器 `code.xedcs.cn`，所以添加不上。
- 用户 GitHub 用户名：`wangmengfan`；目标仓库名：`swift-registry`（文档/README 已改，实际推送需用户执行 git remote）。
- 本次改造：把硬编码的电费计算插件改为标准 `config_flow` + `options_flow` 架构，支持网页 UI 添加与配置管理。
  - 新增 `const.py`、`config_flow.py`、`translations/zh-Hans.json`、`translations/en.json`
  - 重写 `__init__.py`（async_setup_entry/unload_entry + 更新监听）与 `sensor.py`（改用 ConfigEntry 配置项，不再硬编码 SOURCE_SENSOR_ID）
  - `manifest.json` 已加 `"config_flow": true`，`codeowners` 改为 `@wangmengfan`，`documentation` 改为 `https://github.com/wangmengfan/swift-registry`
- 兼容性注意：`hacs.json` 声明 `homeassistant: 2023.1.0`，故未使用 `ConfigFlow(domain=...)` 关键字；`entry.data | entry.options` 用 `dict(...)` 包裹避免 MappingProxyType 不支持 `|`。
- 踩坑：config_flow.py 不要从 `homeassistant.config_entries` 导入 `ConfigFlowResult`（旧版 HA 无此类型），否则模块导入失败、HA 报 `Invalid handler specified`。已移除该导入及返回注解。
- 用户偏好：组件要支持 UI 管理（消除「此设备无法通过用户界面添加」）。
